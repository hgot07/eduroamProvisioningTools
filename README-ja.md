# eduroamProvisioningTools
## 概要
Wi-Fiプロファイルを用いて、eduroamの接続設定を容易にするための、プロファイル生成ツールとコード例です。機関の構成員(教職員や学生)向けのウェブサイトで、ログインが必要な場所に組み込むことで、プロファイル発行サービスを提供できるようになります。

標準では、eduroamのアカウントを使ってOpenRoamingに接続する方法に対応していませんが、コードの変更は容易でしょう (OpenRoamingのRCOIを足せばよい)。

認証方式は、EAP-TTLSとEAP-TLSのみ対応しています。
いくつかのOSは、EAP-TLSをサポートしていないことがあります。

**プロファイルを用いることで、利用者はID・パスワードや各種設定値を手動設定する必要がなくなり、容易かつ正確なeduroam接続設定が可能になります。**
 
**偽基地局を回避するための「サーバ認証」の機能を、確実に設定することができます。**  以下のような問題も減らすことができます。
- 学内では接続できるのによそでは接続できない、または、この逆のケース。(手動設定でTOFU (Trust on First Use)が使われ、端末がWPA3専用モードに設定されることがあり、PMF非対応のWPA2の基地局に接続できなくなる)
- 認証サーバの証明書が更新されると、ネットワーク接続に失敗するようになり、再設定が必要になる。 (手動設定のTOFUでは、Root CAではなく、サーバ証明書のハッシュ値が端末に記録される。このため、証明書が変更されるとサーバ認証に失敗するようになる)

以下の論文でも詳しく紹介しています。
- 後藤英昭, 原田寛之, 漆谷重雄, ``Wi-Fiプロファイルを用いたeduroam/OpenRoamingのパスワードレス設定,'' 学術情報処理研究,  27巻, 1号, pp.197-204, 2023. [https://doi.org/10.24669/jacn.27.1_197](https://doi.org/10.24669/jacn.27.1_197)

## 重要な更新の履歴
[2026/7/15] Perlのモダンな書き方で全面的に刷新されました。
一部のスクリプトのファイル名が変更になっています。
新しい設定ファイルは etc/config.ini になります。
Redisサーバが不要になりました。  
**(重要) config.ini の HelperSecret を適当なランダム文字列に
書き換えてから使ってください。**

[2026/6/20] eduroam CATの4年以上前の古いアプリにバグがあり、
.eap-config の中にワークアラウンドが仕込まれていましたが、これを
廃止しました。具体的には、誤ったタグ \<Username\> しか読み込めない
アプリがあり、正しい \<UserName\> と両方をプロファイルに含めていました。
最近の geteduroam で検証が厳しくなり、不正なプロファイルと判断される
ようになったことから、コミュニティで廃止が支持されました。

[2026/6/20] 2026/1にリリースされた geteduroam では、\<UserName\> と
\<Password\> が埋め込まれたプロファイル (.eap-config) であっても
ID/パスワード入力のプロンプトが表示されるというバグがありました。
このため、自動設定ができませんでした。2026/6/19に修正されたとのことです。
Google Play や F-Droid で公開されるまで、若干の日数がかかります。
